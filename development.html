API 서버 개발

API = Application Programming Interface (클라이언트 앱과 서버 간의 통신 규약) API 방식 : SOAP / RESTful(대부분 사용) , 데이터 포맷은 JSON이나 XML 사용 가능

  1. 인터페이스 설계

    2. 서버와 앱이 주고 받을 데이터와 인터페이스 설계해야함
    인터페이스 목록이 정리되면 서버 개발과 앱 개발 진행
    # 앱 개발 : 클라이언트 개발 or 프론트엔드 개발 / 서버 개발 : 백엔드 개발

  2. DB 설계

    3. DB : RDB(=관계형 데이터베이스) / NoSQL
    RDB : 오라클, MySQL, MariaDB, PostgreSQL 등
    NoSQL : Mongo DB, Couchbase, Cassandra, HBASE 등
    관리할 데이터에 맞게 테이블 생성하고 정규화 작업 # 큰 규모의 프로젝트의 경우 DB전문가인 DBA의 도움을 받을 수 있음

  3. 개발 환경 세팅 및 개발 진행

    4. [RESTful API 서버 프레임워크의 종류]
    구축형 솔루션 : Sinatra, Grape, Rails
    설치형 솔루션 : LoopBack
    클라우드 솔루션 : Parse, Firebase

앱 개발

앱 개발 목표 : 최대한 빠르게 프로토타입을 만드는 것 프로토타입은 간단한 화면을 만들고 타겟 디바이스에 빌드된 앱을 올려 API 서버와 데이터를 주고 받는 모듈 하나를 빠르게 완성하고 테스트 이를 통해 성능이나 다른 이슈가 있지는 확인

  1. 네이티브앱

    2. 구글의 안드로이드 스튜디오로 안드로이폰 개발, 애플의 XCode를 사용하여 아이폰용앱 개발

  2. 모바일웹앱

    3. DB : RDB(=관계형 데이터베이스) / NoSQL
    HTML5, CSS3 웹표준에 따른 웹으로 개발, 디자인과정외에 퍼블리싱 작업 필요, 웹브라우저에서 동작

  3. 하이브리드앱

    4. 기본 기능은 HTML5, CSS5 웹표준으로 개발하고, 패키징은 아이폰, 안드로이드 등 모바일 운영체제별로 구현하는 방식
    개발방식은 Phone Gap, Cordova, ionic, react native 등의 개발플렛폼이 있으며 최근에는 react native 많이 사용
